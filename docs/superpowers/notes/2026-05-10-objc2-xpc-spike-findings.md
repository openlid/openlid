# objc2 NSXPC Spike — Findings

**Date:** 2026-05-10
**Status:** SUCCESS — end-to-end round-trip works
**Branch:** `feat/mvp-implementation`
**Input to:** Task 14 (`crates/helper` NSXPC listener) and Task 20 (`crates/app` NSXPC client)
of `docs/superpowers/plans/2026-05-10-open-lid-mvp.md`.

The spike built two binaries — `spike-server` (LaunchAgent-loaded NSXPC listener
on Mach service `io.openlid.spike`) and `spike-client` — and proved that the
current `objc2` ecosystem can drive a full NSXPC client/server round-trip in
pure Rust **with one assist from Clang**: the XPC protocol must be declared in
a small `.m` file compiled via a `cc` build script. The Rust call to
`spike-client` printed `pong` and exited 0; no NSXPC errors appeared in the
unified log.

This document captures everything Tasks 14 / 20 need.

Tested against:

- **Platform:** macOS 25.3.0 (Darwin), Apple Silicon, Apple clang 17.0.0
- **Cargo workspace** (`spikes/xpc-hello/Cargo.toml`):

```toml

resolver = "2"

[workspace.dependencies]
objc2 = { version = "0.6", features = ["exception"] }   # → resolves to 0.6.4
objc2-foundation = { version = "0.3", features = [      # → resolves to 0.3.2
    "NSXPCConnection",
    "NSString",
    "NSError",
    "NSRunLoop",
    "NSObject",
    "block2",
] }
block2 = "0.6"                                          # → resolves to 0.6.2
```

Notes:
- The single feature flag `NSXPCConnection` on `objc2-foundation` pulls in all
  the XPC types we need: `NSXPCConnection`, `NSXPCListener`, `NSXPCInterface`,
  `NSXPCListenerDelegate` (the trait), `NSXPCListenerEndpoint`,
  `NSXPCConnectionOptions`. The original plan listed each as a separate
  feature; that does no harm but is unnecessary.
- The `objc2` `exception` feature pulls in `objc2-exception-helper` and is
  what makes `objc2::exception::catch` available. Without it, an NSException
  becomes "fatal runtime error: Rust cannot catch foreign exceptions" with
  no context — extremely painful to debug. **Always enable this in the helper
  and client crates.**
- The `block2` feature on `objc2-foundation` is required for the `DynBlock`
  references in the generated `NSXPCConnection` API (`remoteObjectProxyWithErrorHandler:`
  etc.).

**Production workspace Cargo.toml** should add the same three deps with the
same features (verbatim) plus our protocol crate.

## 2. Pattern used

**A combination of three pieces:**

1. **A tiny `.m` file declaring the Objective-C protocol**, compiled into a
   static library via a `build.rs` using the `cc` crate. NSXPC requires the
   protocol's extended method-type metadata to be emitted by Clang, so the
   `extern_protocol!` macro alone is not sufficient.
2. **`define_class!` macro** to create the `Exported` NSObject subclass that
   implements the protocol method and the `ListenerDelegate` NSObject subclass
   that implements `NSXPCListenerDelegate`. Both are generated by `objc2`'s
   macros — no manual class registration.
3. **`block2::RcBlock::with_encoding::<_, _, _, ManualBlockEncoding>(...)`**
   for any block we pass as a reply handler **into** NSXPC. `RcBlock::new`
   produces blocks without the `BLOCK_HAS_SIGNATURE` flag, which NSXPC
   rejects.

**What we did NOT use (and why):**
- `extern_protocol!`: this creates a Rust trait that mirrors an existing
  Obj-C protocol, but it does not register a new protocol with the runtime
  in a way NSXPC accepts. Useful for `NSXPCListenerDelegate` (which is a
  pre-existing system protocol) — and indeed `objc2-foundation` already
  defines that trait via `extern_protocol!` so we just `impl` it in
  `define_class!`.
- `ProtocolBuilder` + `protocol_addMethodDescription`: tried first. NSXPC
  throws `NSInvalidArgumentException: "NSXPCInterface: Unable to get extended
  method signature from Protocol data ... Use of clang is required for
  NSXPCInterface."` See section 7 below.
- Bare `RcBlock::new`: tried for reply blocks; NSXPC throws
  `"Block was not compiled using a compiler that inserts type information
  about arguments."` See section 6.

## 3. Server `main.rs` (complete, working)

The actual file is `spikes/xpc-hello/server/src/main.rs`. For Task 14, copy
this verbatim and apply the `REPLACE for Task 14` comments inline.

```rust
//! spike-server — an NSXPC listener that answers `pingWithReply:` with "pong".
//!
//! Architecture:
//! 1. Reference the `OpenLidSpikeProtocol` Objective-C protocol that lives in
//!    the `spike-protocol` crate. NSXPC requires the protocol metadata be
//!    emitted by Clang (not synthesized at runtime), so the protocol is
//!    declared in a tiny `.m` file compiled by `cc` in that crate's build.rs.
//! 2. Define an `Exported` NSObject subclass that implements the protocol
//!    method `pingWithReply:` — when invoked, calls the reply block with "pong".
//! 3. Define a `ListenerDelegate` NSObject subclass that conforms to
//!    `NSXPCListenerDelegate` and accepts incoming connections, wiring each one
//!    with the exported interface and a fresh `Exported` instance.
//! 4. Create `NSXPCListener::initWithMachServiceName("io.openlid.spike")`, set
//!    the delegate, `resume()`, and run the run loop forever.
//!
//! REPLACE for Task 14 (production helper):
//! - Replace `spike-protocol` with a production protocol crate (or inline its
//!   .m / .h files inside `crates/helper`) that declares
//!   `@protocol OpenLidHelperProtocol` with three methods:
//!     - setSleepPreventionEnabled:withReply:
//!     - getSleepPreventionStatusWithReply:
//!     - pingWithReply:
//! - Each `Exported`-side method dispatches into a `HelperImpl<P: Pmset>`.
//! - In `listener_shouldAcceptNewConnection`, before returning true, call
//!   `connection.setCodeSigningRequirement(&team_id_req)` to enforce the
//!   client's signing identity.

use std::ptr::NonNull;

use block2::Block;
use objc2::rc::Retained;
use objc2::runtime::{NSObjectProtocol, ProtocolObject};
use objc2::{define_class, msg_send, AnyThread};
use objc2_foundation::{
    NSObject, NSRunLoop, NSString, NSXPCConnection, NSXPCInterface, NSXPCListener,
    NSXPCListenerDelegate,
};

define_class!(
    /// The object exported on each accepted XPC connection. Implements the
    /// `OpenLidSpikeProtocol` declared in `spike-protocol`.
    ///
    /// REPLACE for Task 14: add ivars `Arc<HelperImpl<P>>` and dispatch the
    /// three setSleepPrevention/getStatus/ping methods into it.
    #[unsafe(super(NSObject))]
    #[name = "OpenLidSpikeExported"]
    struct Exported;

    impl Exported {
        // The reply block is passed by Objective-C as a non-null pointer to
        // a Block. We invoke it once with an NSString containing "pong".
        //
        // REPLACE for Task 14: dispatch into a real `HelperImpl` method.
        #[unsafe(method(pingWithReply:))]
        fn ping_with_reply(&self, reply: NonNull<Block<dyn Fn(NonNull<NSString>)>>) {
            let pong = NSString::from_str("pong");
            unsafe {
                let pong_ptr = NonNull::from(&*pong);
                reply.as_ref().call((pong_ptr,));
            }
        }
    }

    unsafe impl NSObjectProtocol for Exported {}
);

impl Exported {
    fn new() -> Retained<Self> {
        let this = Self::alloc().set_ivars(());
        unsafe { msg_send![super(this), init] }
    }
}

define_class!(
    /// NSXPCListenerDelegate implementation. Holds no state — for every new
    /// connection, builds an `NSXPCInterface` from our protocol, creates a
    /// fresh `Exported`, wires them together, and resumes the connection.
    ///
    /// REPLACE for Task 14: in `listener_shouldAcceptNewConnection`, before
    /// returning true, call `connection.setCodeSigningRequirement(&team_id_req)`
    /// to enforce that only correctly-signed clients can connect.
    #[unsafe(super(NSObject))]
    #[name = "OpenLidSpikeListenerDelegate"]
    struct ListenerDelegate;

    impl ListenerDelegate {}

    unsafe impl NSObjectProtocol for ListenerDelegate {}

    unsafe impl NSXPCListenerDelegate for ListenerDelegate {
        #[unsafe(method(listener:shouldAcceptNewConnection:))]
        fn listener_should_accept_new_connection(
            &self,
            _listener: &NSXPCListener,
            new_connection: &NSXPCConnection,
        ) -> bool {
            eprintln!("[spike-server] accepting new connection");
            // SAFETY: protocol metadata is generated by Clang in spike-protocol
            // and matches our exported object's method signatures.
            let interface = unsafe {
                NSXPCInterface::interfaceWithProtocol(spike_protocol::protocol())
            };
            new_connection.setExportedInterface(Some(&interface));

            let exported = Exported::new();
            // SAFETY: `exported` conforms to OpenLidSpikeProtocol via define_class!.
            unsafe {
                new_connection.setExportedObject(Some(&*exported));
            }

            new_connection.resume();
            true
        }
    }
);

impl ListenerDelegate {
    fn new() -> Retained<Self> {
        let this = Self::alloc().set_ivars(());
        unsafe { msg_send![super(this), init] }
    }
}

fn main() {
    eprintln!("[spike-server] starting (pid={})", std::process::id());

    let service_name = NSString::from_str("io.openlid.spike");
    let listener =
        NSXPCListener::initWithMachServiceName(NSXPCListener::alloc(), &service_name);

    // The listener holds the delegate as a weak reference, so we must keep
    // `delegate` alive for the lifetime of the process. `main` runs the
    // run loop forever, so leaking via `Box::leak` would also work; here we
    // just bind to a stack variable that outlives `run_loop.run()`.
    let delegate = ListenerDelegate::new();
    let delegate_proto: &ProtocolObject<dyn NSXPCListenerDelegate> =
        ProtocolObject::from_ref(&*delegate);
    listener.setDelegate(Some(delegate_proto));

    listener.resume();

    eprintln!("[spike-server] listener resumed; entering run loop");

    let run_loop = NSRunLoop::currentRunLoop();
    run_loop.run();
}
```

## 4. Client `main.rs` (complete, working)

The actual file is `spikes/xpc-hello/client/src/main.rs`.

```rust
//! spike-client — connects to `io.openlid.spike`, calls `pingWithReply:`,
//! prints the reply, and exits.
//!
//! REPLACE for Task 20 (production menubar client):
//! - Three protocol method descriptions instead of one (set/get/ping).
//! - Helper functions that bridge each protocol method to a sync Rust API.
//!   The pattern is the same as `call_ping` below: build a reply block, send
//!   the message, recv() the result.

use std::ffi::CStr;
use std::process::ExitCode;
use std::ptr::NonNull;
use std::sync::mpsc;
use std::time::Duration;

use block2::{ManualBlockEncoding, RcBlock};
use objc2::exception::catch;
use objc2::msg_send;
use objc2::rc::Retained;
use objc2::runtime::AnyObject;
use objc2::AnyThread;
use objc2_foundation::{
    NSError, NSString, NSXPCConnection, NSXPCConnectionOptions, NSXPCInterface,
};

/// Block encoding for `void (^)(NSString *)`. This is the type that NSXPC
/// requires the reply block to advertise via its embedded signature. We
/// supply it manually because `block2::RcBlock::new` constructs blocks
/// without the `BLOCK_HAS_SIGNATURE` flag set, and NSXPC then refuses to
/// invoke them.
///
/// REPLACE for Task 20: define one encoding struct per reply block signature.
/// See section 9 for the three encodings the helper protocol needs.
struct ReplyBlockEncoding;
// SAFETY: encoding string matches the block's argument list `(NonNull<NSString>,)`
// and return type `()`. On 64-bit Apple platforms this is `v16@?0@8`.
unsafe impl ManualBlockEncoding for ReplyBlockEncoding {
    type Arguments = (NonNull<NSString>,);
    type Return = ();
    const ENCODING_CSTR: &'static CStr = c"v16@?0@8";
}

/// Issue `pingWithReply:` against the given XPC connection and block until the
/// reply (or error / timeout) arrives.
///
/// REPLACE for Task 20: copy this shape three times (one per helper method),
/// each with its own reply-block signature and result type. e.g.
/// `call_set_sleep_prevention(conn, true) -> Result<(), HelperError>`.
fn call_ping(conn: &NSXPCConnection) -> Result<String, String> {
    let (tx, rx) = mpsc::channel::<Result<String, String>>();

    // Error handler for `remoteObjectProxyWithErrorHandler:`. Fires only if
    // the proxy itself errors out (e.g. interface mismatch). For ordinary
    // connection problems the `invalidationHandler` set on the connection
    // is what fires.
    let err_tx = tx.clone();
    let err_handler = RcBlock::new(move |err: NonNull<NSError>| {
        let s = unsafe { err.as_ref() }.to_string();
        let _ = err_tx.send(Err(format!("proxy error: {s}")));
    });

    // The reply block — invoked by NSXPC when the server calls the reply.
    //
    // We use `RcBlock::with_encoding` instead of `RcBlock::new` so that the
    // block's descriptor sets BLOCK_HAS_SIGNATURE with our hand-rolled
    // signature string. Without this, NSXPC throws NSInvalidArgumentException:
    // "Block was not compiled using a compiler that inserts type information
    // about arguments".
    let reply_tx = tx;
    let reply_block = RcBlock::with_encoding::<_, _, _, ReplyBlockEncoding>(
        move |msg: NonNull<NSString>| {
            let s = unsafe { msg.as_ref() }.to_string();
            let _ = reply_tx.send(Ok(s));
        },
    );

    let proxy: Retained<AnyObject> = conn.remoteObjectProxyWithErrorHandler(&err_handler);

    // SAFETY: the selector matches the protocol the connection was configured
    // with, the block has the right signature, and the return is void.
    unsafe {
        let _: () = msg_send![&*proxy, pingWithReply: &*reply_block];
    }

    match rx.recv_timeout(Duration::from_secs(5)) {
        Ok(r) => r,
        Err(mpsc::RecvTimeoutError::Timeout) => Err("timed out waiting for reply".to_string()),
        Err(mpsc::RecvTimeoutError::Disconnected) => Err("channel disconnected".to_string()),
    }
}

fn main() -> ExitCode {
    let result = catch(|| run());
    match result {
        Ok(code) => code,
        Err(Some(exc)) => {
            eprintln!("[spike-client] NSException: {}", exc);
            ExitCode::FAILURE
        }
        Err(None) => {
            eprintln!("[spike-client] unknown foreign exception");
            ExitCode::FAILURE
        }
    }
}

fn run() -> ExitCode {
    let service_name = NSString::from_str("io.openlid.spike");
    let conn = NSXPCConnection::initWithMachServiceName_options(
        NSXPCConnection::alloc(),
        &service_name,
        NSXPCConnectionOptions(0),
    );

    // SAFETY: protocol metadata is Clang-emitted in spike-protocol.
    let interface = unsafe { NSXPCInterface::interfaceWithProtocol(spike_protocol::protocol()) };
    conn.setRemoteObjectInterface(Some(&interface));

    let invalidation_block = RcBlock::new(|| {
        eprintln!("[spike-client] connection invalidated by NSXPC");
    });
    conn.setInvalidationHandler(Some(&invalidation_block));

    let interruption_block = RcBlock::new(|| {
        eprintln!("[spike-client] connection interrupted");
    });
    conn.setInterruptionHandler(Some(&interruption_block));

    conn.resume();

    match call_ping(&conn) {
        Ok(s) => {
            println!("{s}");
            conn.invalidate();
            ExitCode::SUCCESS
        }
        Err(e) => {
            eprintln!("[spike-client] error: {e}");
            conn.invalidate();
            ExitCode::FAILURE
        }
    }
}
```

## 7. Gotchas (read this before Tasks 14 and 20)

### 7a. NSXPC needs Clang-emitted protocol metadata, not runtime-registered

**The single biggest finding.** My first attempt registered the protocol
dynamically using `ProtocolBuilder` and `protocol_addMethodDescription`:

```rust
// THIS DOES NOT WORK with NSXPC.
let mut b = ProtocolBuilder::new(c"OpenLidSpikeProtocol").unwrap();
b.add_method_description::<(NonNull<Block<...>>,), ()>(sel!(pingWithReply:), true);
b.register();
```

`NSXPCInterface.interfaceWithProtocol:` then throws:

> `NSInvalidArgumentException: NSXPCInterface: Unable to get extended method
> signature from Protocol data (OpenLidSpikeProtocol / pingWithReply:). Use of
> clang is required for NSXPCInterface.`

The runtime API `protocol_addMethodDescription` writes the basic type string
but not the **extended** type string that NSXPC requires. Clang writes that
extended string when it sees `@protocol Foo ... @end` in source. There is no
public objc-runtime API to add extended types, so a small `.m` file is the
only practical solution.

Confirmed with a pure-Obj-C reproduction in `/tmp/check_proto.m`. The same
NSException occurred there, so this is not a Rust limitation.

### 7b. Reply blocks need a manually-set signature

`block2::RcBlock::new(closure)` produces a block that lacks the
`BLOCK_HAS_SIGNATURE` flag. NSXPC introspects every reply block before
invoking it, and refuses to invoke unsigned blocks with:

> `NSInvalidArgumentException: Block was not compiled using a compiler that
> inserts type information about arguments. (pingWithReply:)`

The fix is `block2::RcBlock::with_encoding::<_, _, _, MyEnc>(closure)`
where `MyEnc: ManualBlockEncoding`. The encoding string follows Apple's
block signature format: `<ret_encoding><total_size>@?0<arg_encoding><offset>...`.
For 64-bit:

| Block signature | Encoding string |
|---|---|
| `void (^)(NSString *)` | `v16@?0@8` |
| `void (^)(void)` | `v8@?0` |
| `void (^)(BOOL, NSString *)` | `v17@?0B8@9` (BOOL is `B`, size 1; pointer at offset 9) |
| `void (^)(BOOL, BOOL, NSString *)` | `v18@?0B8B9@10` |

The size after the return type is the total of (8 for the hidden block self) +
sum of argument sizes. `@` is the Object encoding (size 8 on 64-bit), `B` is
Bool (size 1, but typically aligned/padded — see below).

**Caveat about size/offsets:** Apple's signature parsing is permissive enough
that simple `v16@?0@8` works even with no alignment padding. For mixed
primitive + object arguments (like our production `BOOL+NSString`), the
offsets may need to be padded to natural alignment. **Test each new signature
with a small standalone smoke test before depending on it.** A useful sanity
check: `block2::encoding::block_signature_string` (private) shows what
block2 computes; we can replicate that logic from `block2/src/encoding.rs`.

### 7c. Exception feature is mandatory for debugging

Without `objc2 = { features = ["exception"] }`, any NSException becomes:

> `fatal runtime error: Rust cannot catch foreign exceptions`

with no message, no backtrace, no crash report content. Wrap the entry point
in `objc2::exception::catch(|| run())` and print the exception so you get the
actual `NSException.reason`. This saved 80% of the spike's debug time.

### 7d. `setExportedObject` is `unsafe` and you must use `&*retained`

```rust
let exported = Exported::new();
unsafe { new_connection.setExportedObject(Some(&*exported)); }
```

The `unsafe` is because NSXPC requires the exported object to conform to
the protocol set as `exportedInterface`; Rust can't verify this at compile
time. The `&*exported` deref pattern is needed because `Retained<Exported>`
doesn't auto-coerce to `&AnyObject`.

### 7e. Listener delegate is held weakly

`NSXPCListener.setDelegate:` stores the delegate as a **weak reference**. If
you drop your `Retained<ListenerDelegate>` before the run loop starts, the
listener silently stops accepting connections. The fix in the spike was to
bind the delegate to a local stack variable in `main`; production code
should either do the same or `Box::leak` it (since the listener lives for
the whole process anyway).

### 7f. The reply block reference must use `NonNull<Block<...>>` in the
`#[unsafe(method)]` signature, not `&Block<...>`

This:

```rust
#[unsafe(method(pingWithReply:))]
fn ping_with_reply(&self, reply: NonNull<Block<dyn Fn(NonNull<NSString>)>>) {
```

works. `&Block<...>` would *also* work for the encoding (`Block` is the
encoding for both), but the message dispatcher passes a raw pointer that
could theoretically be NULL for some calls, and `NonNull` is the more
honest type. The block lifetime is implicitly the duration of the call —
NSXPC retains it on the other side if needed.

### 7g. NSString conversion to Rust string

`NSString::to_string()` (from `objc2-foundation`'s `Display`/`ToString`)
allocates a Rust `String` from the NSString's UTF-8 bytes. This is what the
client uses to receive "pong". The conversion is allocating; if performance
matters in production, use `NSString::as_str(&pool)` with an autorelease pool
instead.

### 7h. `AnyThread::alloc()` for NSObject subclasses

Calling `Exported::alloc()` requires `use objc2::AnyThread;` to be in scope.
The trait is named differently than the plan's text might suggest — it's
`AnyThread`, not `AllocAnyThread`. (Renamed somewhere in the 0.6 series.)

### 7i. `objc2_foundation::NSObject` is the right superclass

For both `define_class!` blocks, the superclass is `NSObject` from
`objc2_foundation` (NOT `objc2::runtime::NSObject`, although they alias).
Same import path makes the example self-consistent.

## 9. How to apply this to Tasks 14 and 20

### Task 14: `crates/helper` NSXPC listener

The helper protocol mirrors `open_lid_core::ipc::helper::*`:

```objc
@protocol OpenLidHelperProtocol
- (void)setSleepPreventionEnabled:(BOOL)enabled
                        withReply:(void (^)(BOOL ok, NSString * _Nullable error))reply;
- (void)getSleepPreventionStatusWithReply:(void (^)(BOOL ok, BOOL active, NSString * _Nullable error))reply;
- (void)pingWithReply:(void (^)(void))reply;
@end
```

**Steps (mechanical from the spike):**

1. Create `crates/helper-protocol/` with the same structure as
   `spikes/xpc-hello/protocol/` — `Cargo.toml`, `build.rs`, `objc/*.{h,m}`,
   `src/lib.rs`. Rename `OpenLidSpikeProtocol` → `OpenLidHelperProtocol`.
2. Add the three method declarations above to the `.h` file. Verbatim.
3. In `crates/helper/src/main.rs`, copy the spike-server structure:
   - `define_class!` for `Exported`. Add an ivar `helper: Arc<HelperImpl<P>>`
     (where `P: Pmset`). Override the three method handlers; each delegates
     to `self.ivars().helper`. The match-on-Result→reply-block translation is
     simple: success path calls `reply(true, ...)`, error path calls
     `reply(false, NSString::from_str(&err))`.
   - `define_class!` for `ListenerDelegate`. Same shape as the spike. **Add**
     `new_connection.setCodeSigningRequirement(&NSString::from_str(team_id_req))`
     before `setExportedInterface` to enforce the client's signing identity
     (per spec section "Helper authentication").
   - Main function: `NSXPCListener::initWithMachServiceName("io.openlid.helper")`,
     wire delegate, resume, run loop.
4. The `Exported::ping_with_reply` body should match the helper's response
   policy from the design (just call `reply()`, no arguments).
5. For `setSleepPreventionEnabled:withReply:`, the method signature in Rust
   becomes:
   ```rust
   #[unsafe(method(setSleepPreventionEnabled:withReply:))]
   fn set_sleep_prevention(
       &self,
       enabled: objc2::runtime::Bool,
       reply: NonNull<Block<dyn Fn(objc2::runtime::Bool, *mut NSString)>>,
   ) { ... }
   ```
   Note `*mut NSString` for the nullable error string (use `core::ptr::null_mut()`
   on the success path).
6. The plist becomes a LaunchDaemon (root) — see section 6.

### Task 20: `crates/app` NSXPC client (menubar process)

The `crates/app` menubar role calls the three helper methods. The pattern is
**three copies** of the spike's `call_ping`, parameterized:

1. **One `ManualBlockEncoding` impl per reply signature.** Three total:
   - `Reply_OkErr` for `void(^)(BOOL, NSString *)` — encoding `v17@?0B8@9`
     (or `v24@?0B8@16` if natural alignment is enforced — test).
   - `Reply_OkActiveErr` for `void(^)(BOOL, BOOL, NSString *)` — encoding
     `v18@?0B8B9@10` (or padded `v24@?0B8B9@16` — test).
   - `Reply_Empty` for `void(^)(void)` — encoding `v8@?0`.

   When in doubt, build a minimal Clang-compiled .o, dump its block layout
   with `otool -tv`, and copy the exact string Clang emitted.

2. **Three Rust helper functions** that follow the spike's `call_ping`
   pattern. Sketch:

   ```rust
   pub fn call_set_sleep_prevention(
       conn: &NSXPCConnection, enabled: bool,
   ) -> Result<(), HelperError> {
       let (tx, rx) = mpsc::channel();
       let err_h = RcBlock::new(move |err: NonNull<NSError>| {
           tx.send(Err(HelperError::Proxy(unsafe { err.as_ref() }.to_string()))).ok();
       });
       let tx2 = tx.clone();
       let reply = RcBlock::with_encoding::<_, _, _, ReplyOkErr>(
           move |ok: Bool, err: *mut NSString| {
               if ok.as_bool() {
                   tx2.send(Ok(())).ok();
               } else {
                   let msg = if err.is_null() {
                       "unknown error".to_string()
                   } else {
                       unsafe { &*err }.to_string()
                   };
                   tx2.send(Err(HelperError::Helper(msg))).ok();
               }
           },
       );
       let proxy = conn.remoteObjectProxyWithErrorHandler(&err_h);
       unsafe {
           let _: () = msg_send![
               &*proxy,
               setSleepPreventionEnabled: Bool::new(enabled),
               withReply: &*reply
           ];
       }
       rx.recv_timeout(Duration::from_secs(5))
           .map_err(|_| HelperError::Timeout)?
   }
   ```

3. **Connection setup is shared** — one `NSXPCConnection` for the lifetime of
   the menubar process. Set `setCodeSigningRequirement` to pin the helper's
   identity (security spec), wire invalidation/interruption handlers, then
   call the three functions on demand.

4. **All XPC calls block via `mpsc`.** The menubar's UI loop calls these from
   a worker thread (or from `dispatch_async`), never from the AppKit main
   thread, because they sleep up to `recv_timeout`.

## 11. TL;DR for the impatient implementer

1. **Three crates** in the workspace, not two. Add a tiny `crates/helper-protocol`
   that contains a Clang-compiled `.m` declaring `@protocol OpenLidHelperProtocol`.
2. **`objc2`, `objc2-foundation`, `block2`** at the versions in section 1, with
   the `exception` feature enabled on `objc2`.
3. **Reply blocks built with `RcBlock::with_encoding`**, never `RcBlock::new`.
4. **Wrap entry points in `objc2::exception::catch(...)`** so NSExceptions surface.
5. **Helper plist is a LaunchDaemon (root)**, but the objc2 code is identical
   to this spike's LaunchAgent version.
